00049 // ---- Check Sum ----
00050 /* The Checksum that is used is a rolling 8-bit sum from the [From] to the last parameter byte of a command
00051    packet and from the [Length] to the last data byte of a Data packet. This sum is then 1-complemented
00052    (~, not) and passed as [Chk]. This prevents a series of 0x00 replys from passing the correct check sum. 
00053    Because of the inversion, a packet with all zeros should have a checksum of 0xFF.
00054    
00055    The other nice feature of this checksum, is that no matter what the data is, if you add the checksum
00056    ([Chk]) to the final sum, you should get 0xFF.
00057    
00058    To make it even cleaner, you can start the rolling checksum at 0x01 such that when you add in all of the
00059    data bytes and the [Chk] byte, you get 0x00. This effectively makes the whole operation a twos complement
00060 */
00061 #define EDP_CHECKSUM_INIT 0x01
